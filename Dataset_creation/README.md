# Text-in-Image Dataset Creation Pipeline — v12

> **OCR:** EasyOCR &nbsp;|&nbsp; **Captions:** BLIP-2 &nbsp;|&nbsp; **Source:** [stzhao/AnyWord-3M](https://huggingface.co/datasets/stzhao/AnyWord-3M)

---

## Overview

`create_dataset_cluster.py` builds a **text-in-image dataset** suited for training text-to-image generative models (e.g. Stable Diffusion, ControlNet). For each accepted image the pipeline outputs:

| Field | Description |
|---|---|
| `text` | The verified, readable text found in the image |
| `caption` | A general visual description generated by BLIP-2 |
| `prompt` | A training sentence that embeds the OCR text verbatim |

Images are drawn from **AnyWord-3M** (HuggingFace, streamed by default — no full download needed).

---

## Pipeline — Step by Step

```
AnyWord-3M stream
      │
      ▼
1. Annotation filtering      → keep valid, short, English text regions
      │
      ▼
2. Image quality gate        → resolution / sharpness / brightness / contrast
      │
      ▼
3. EasyOCR verify            → confirm the annotated text is readable in the image
      │
      ▼
4. EasyOCR reconstruct       → expand to the full visible phrase near the bbox
      │
      ▼
5. BLIP-2 caption            → general visual description (no OCR text forced)
      │
      ▼
6. BLIP-2 training prompt    → natural sentence with OCR text in quotation marks
      │
      ▼
7. Save                      → train / val / test split + JSON files
```

## Quality Thresholds

### Image quality
| Parameter | Value |
|---|---|
| Minimum resolution | 256 px (each side) |
| Minimum sharpness (Laplacian variance) | 80 |
| Brightness range | 30 – 230 |
| Minimum contrast (std dev) | 20 |

### Annotation text filters
| Parameter | Value |
|---|---|
| Min / max text length | 1 – 40 characters |
| Max words per annotation | 5 |
| Must contain Latin characters | Yes |
| Non-ASCII characters | Rejected |

### Dense-text rejection
| Parameter | Value |
|---|---|
| Max valid text regions per image | 4 |
| Max total characters across all annotations | 40 |

### EasyOCR verification
| Parameter | Value |
|---|---|
| Min EasyOCR confidence | 0.40 |
| Min Levenshtein coverage score | 0.30 |
| Min bounding-box height | 30 px |

### Phrase reconstruction
| Parameter | Value |
|---|---|
| Max phrase words | 10 |
| Max phrase characters | 80 |

## Dataset Subsets Used

| Subset | Type |
|---|---|
| `laion` | Web-crawled natural images |
| `OCR_COCO_Text` | COCO scene text |
| `OCR_mlt2019` | Multi-language (English filtered) |
| `OCR_Art` | Artistic / designed text |


---

## Caption Models

| Flag | Model | VRAM required | Notes |
|---|---|---|---|
| `--caption-model blip2` *(default)* | `Salesforce/blip2-flan-t5-xl` | ~16 GB | Best quality; falls back to BLIP if load fails |
| `--caption-model blip` | `Salesforce/blip-image-captioning-large` | ~6 GB | Faster, lighter |

## Usage

### Basic run
```bash
python create_dataset_cluster.py
```

### Full options
```bash
python create_dataset_cluster.py \
    --output        dataset_output \   # output directory
    --max-images    1000 \             # number of accepted images to collect
    --subsets       laion OCR_COCO_Text OCR_mlt2019 OCR_Art \
    --caption-model blip2 \            # blip2 (default) or blip
    --no-streaming                     # use only if dataset is pre-downloaded
```

### Run on SLURM cluster
```bash
sbatch job_creation_dataset.sh
```

## Hardware Requirements

| Resource | Minimum | Recommended |
|---|---|---|
| GPU | 1× GPU with 16 GB VRAM (for BLIP-2) | L40S, A100, V100 |
| CPU cores | 4 | 8 |
| RAM | 32 GB | 48 GB |
| Disk | 10 GB (streaming) | 250 GB (full download) |
| Time (1 000 images) | ~2 h | ~1 h on L40S |

> **Note:** BLIP-2 (`blip2-flan-t5-xl`) requires ~16 GB VRAM. If your GPU has less, use `--caption-model blip` (~6 GB VRAM).

## Troubleshooting

**BLIP-2 fails to load**
→ The pipeline automatically falls back to `blip-image-captioning-large`. This is normal on GPUs with < 16 GB VRAM.

**`No dataset subsets could be loaded`**
→ Check your internet connection or HuggingFace token (`HF_HOME`, `HUGGING_FACE_HUB_TOKEN`).

**Very low acceptance rate**
→ Expected. AnyWord-3M contains many non-English or low-quality images. The pipeline typically accepts 1 in 5–15 images depending on the subset.

**CUDA out of memory**
→ Switch to `--caption-model blip`, or reduce batch concurrency by using fewer SLURM CPU cores.
